# Challenge, pacing, and level design

## Challenge model

Challenge emerges from:

- perception;
- decision;
- execution;
- planning/resource management;
- adaptation.

Identify which layer the mechanic tests. Increasing speed often raises perception and execution difficulty while reducing meaningful decision time.

## Difficulty curve

Introduce one dimension at a time:

1. teach in safety;
2. confirm alone;
3. combine with a known mechanic;
4. add time/space pressure;
5. vary context;
6. test mastery;
7. provide recovery/rest.

Use difficulty spikes deliberately and telegraph rule changes.

## Pacing

Alternate:

- anticipation;
- execution;
- payoff;
- recovery;
- navigation/exploration;
- escalation.

Constant intensity becomes flat. Quiet spaces establish contrast and allow learning.

## Encounter readability

Define roles:

- pressure/approach;
- denial/area control;
- ranged threat;
- support;
- reward/objective;
- environmental hazard.

Limit simultaneous new information. Use spawn direction, silhouette, movement, sound, and color to distinguish roles.

## Level composition

Guide with:

- landmarks;
- path contrast;
- framed vistas;
- light and color;
- enemy/reward placement;
- elevation;
- occlusion and reveal;
- repeated motifs.

Avoid arrows as the only navigation system. The world should explain itself.

## Failure and recovery

Failure should be:

- attributable;
- anticipated enough to learn;
- proportionate;
- quick to recover from;
- consistent with rules.

Restart near the decision being practiced. Remove stale entities, timers, VFX, physics, and input state.

## Tuning evidence

Track completion time, failure location, repeated confusion, unused verbs, camera corrections, and restart delay. Qualitative observation matters more than one aggregate score.
